// 01 — Sobre

Quem Sou

Sou Pedro Henrique Maciel, desenvolvedor backend baseado no Rio de Janeiro, Brasil.

Vim da engenharia civil e da gestão de segurança do trabalho em ambiente hospitalar — onde aprendi que sistemas mal projetados têm consequências reais. Trago esse mesmo rigor para o software: cada decisão de arquitetura importa, cada ponto de falha precisa ser previsto.

Minha stack principal é Java + Spring Boot, com experiência sólida em APIs REST, PostgreSQL e autenticação JWT. Tenho também background em Node.js e TypeScript pelo frontend do meu projeto principal em produção.

Me importo com código que é fácil de ler, testar e manter — não apenas código que funciona hoje. Atualmente expandindo para DevOps, Docker e infraestrutura cloud.

Finance API

SaaS · Multi-Tenant · Em Produção ↗ Ver Sistema ● Ativo

Finance API — SaaS Financeiro

Multi-Tenant · JWT + Refresh Token · Audit Log · Bucket4j · LGPD · DRE · Testcontainers

Backend → Frontend →

SaaS financeiro multi-tenant de nível produção com isolamento completo de dados por empresa via TenantContext thread-local, autenticação JWT com refresh token, log de auditoria em 22 ações distintas, rate limiting sem Redis (Bucket4j), conformidade LGPD (DELETE /me), DRE, conciliação bancária e importação via CSV/XLSX. 87 commits, testes de integração com Testcontainers e CI/CD via GitHub Actions.

// 05 — PGEO · SESMT

PGEO — SESMT

Sistema Real em Produção Hospital · +1.000 funcionários ● Ativo

PGEO — Gestão de Saúde Ocupacional

SESMT Hospitalar · Spring MVC · Thymeleaf · WebSocket STOMP · Docker · nginx SSL

GitHub →

Sistema web completo de gestão do SESMT de um hospital com mais de 1.000 funcionários. Gerencia agendamentos de exames ocupacionais e ASO com atualizações em tempo real via WebSocket STOMP, atestados médicos com controle de absenteísmo (alerta automático INSS), calendário semanal visual, geração de documentos em PDF e Excel, e trilha de auditoria completa de todas as ações.
